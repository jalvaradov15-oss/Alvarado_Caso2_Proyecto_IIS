# **Investigación: Markdown, su uso en proyectos de software y su relación con GitHub**

#### **1. ¿Qué es Markdown?**



Markdown es un lenguaje de marcado ligero creado en 2004 por John Gruber y Aaron Swartz. Su objetivo es permitir que un documento escrito en texto plano sea claro y fácil de leer, pero que también pueda convertirse a HTML u otros formatos.



Utiliza símbolos simples —como # para títulos, \* para énfasis o - para listas— para aplicar formato sin depender de HTML o editores avanzados. Su fortaleza está en que el texto sigue siendo comprensible incluso sin procesarlo.



#### **2. ¿Por qué se usa Markdown en proyectos de software?**



Markdown se ha convertido en una herramienta fundamental en el desarrollo por varias razones:



a) Simplicidad y rapidez



Permite escribir documentación de manera ágil sin aprender lenguajes más complejos. Es ideal para guías, instrucciones y notas técnicas.



b) Integración con control de versiones



Al ser texto plano, Git puede mostrar diferencias, comparar versiones y resolver conflictos con precisión. Esto lo hace muy adecuado para trabajo colaborativo.



c) Estándar para documentación



Markdown se emplea en archivos como:



README.md



CONTRIBUTING.md



LICENSE.md



CHANGELOG.md



Manuales internos y documentación técnica



Por su uso extendido, se considera casi un estándar dentro de los repositorios.



#### **3. Markdown y GitHub: ventajas y desventajas**



GitHub utiliza una variante llamada GitHub Flavored Markdown (GFM), que añade funciones útiles específicamente para proyectos colaborativos.



3.1 Ventajas de usar Markdown en GitHub



* Integración automática

GitHub muestra los archivos .md con formato atractivo sin necesidad de programas adicionales.



* Funciones adicionales con GFM

Incluye tablas, listas de tareas, resaltado de código, menciones y referencias automáticas a issues y pull requests.



* Perfecto para control de versiones

Git muestra los cambios línea por línea, lo que facilita revisar y entender modificaciones.



* Legible incluso sin formato

Un archivo Markdown sigue siendo claro incluso si se abre en un editor básico como Bloc de notas.



3.2 Desventajas de usar Markdown en GitHub



* Limitaciones de diseño

Markdown no permite crear estructuras visuales complejas ni contenido interactivo avanzado.



* Diferencias entre plataformas

Distintos servicios interpretan Markdown de forma ligeramente distinta, lo que puede causar variaciones en su apariencia.



* Dependencia del renderizado de GitHub

La visualización más estética se obtiene dentro de GitHub; en otros entornos se verá como texto plano con su sintaxis.



Conclusión



Markdown es una herramienta esencial para documentar proyectos de software gracias a su simplicidad, legibilidad y compatibilidad con Git. Su integración con GitHub mediante GFM lo convierte en una herramienta poderosa para la documentación colaborativa y la organización de proyectos.
